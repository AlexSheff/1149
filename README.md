## Architecture — From Core to Periphery

The repository is structured from ethical foundation to physical implementation.  
Every collaborator accepts the **Ethical OS** first, then proceeds to assembly.

```
01_Ethical_OS        — WHY: The license agreement. Ethics, trust, principles.
02_Social_Protocol   — The global interface for human interaction.
03_Meta_Ecosystems   — Primary environments for consciousness defragmentation.
04_Technical_Nodes   — Tools for digital sovereignty, AI, cryptography.
05_Reality_Hardware  — Physical and economic interfaces with the world.
06_Social_Impact     — Communities, cultural filters, solidarity in practice.
07_Research_Lab      — Future hypotheses and speculative research.
08_Archive           — Personal logbook and verified historical record.
```

**The logic:** Why → Environment → Tools → Physical World.

## Document Types

Each project folder contains one or both:
- `WHITE_PAPER.md` — The *why*: philosophy, vision, social architecture, mechanism
- `TECH_SPEC.md` — The *how*: architecture, stack, algorithms, data models, code

## Solidarity Check

Every system that touches economics must answer:  
> **"Where does solidarity happen structurally?"**

## How to Add an Idea

1. Pick the right layer folder
2. Copy `_templates/WHITE_PAPER_TEMPLATE.md` or `TECH_SPEC_TEMPLATE.md`
3. Fill in the fields
4. The site at `AlexSheff.github.io/1149` reads from `data/ideas.json`
