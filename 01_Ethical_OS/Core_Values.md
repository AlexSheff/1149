# Core Values — Peace, Justice, Equality, Solidarity

> *Four words. Each one is a design constraint, not a goal.*

---

## 1. Peace

Peace is not the absence of conflict. It is the presence of **just structures** that make violence unnecessary.

Systems built under this value must:
- Reduce conditions that generate conflict, not merely suppress symptoms
- Create mechanisms for disagreement that don't require domination
- Reward de-escalation structurally (see: Guardians app)

---

## 2. Justice

Justice is **procedural integrity** — consistent, transparent, and auditable processes.

Systems built under this value must:
- Apply the same rules to all participants regardless of status
- Make their decision-making processes inspectable by those affected
- Have explicit, documented appeal mechanisms

---

## 3. Equality

Equality of **access** and **voice** — not equality of outcome.

Systems built under this value must:
- Not create structural advantages based on pre-existing wealth or connections
- Allow meaningful participation without capital investment
- Weight governance by contribution, not by ownership (see: Choose&Own stake decay)

---

## 4. Solidarity

Solidarity is the structural embedding of mutual care.

> If your system requires one person to voluntarily help another, you have not built solidarity — you have built charity. Solidarity is when the system makes extraction impossible.

Systems built under this value must:
- Embed redistribution at the protocol level, not the policy level
- Make the solidarity mechanism auditable and automatic (see: Fair BEAT)
- Ensure that value created by the community accrues to the community

---

## 5. Implementation Test

For every system, run this check:

| Question | Pass | Fail |
|----------|------|------|
| Can a wealthy actor extract value and block redistribution? | No | Yes → redesign |
| Can someone participate without pre-existing resources? | Yes | No → redesign |
| Is the redistribution mechanism automatic? | Yes | No → redesign |
| Are the rules auditable by participants? | Yes | No → redesign |
