# Contributing to Reality Refactor Lab

---

## For Founder

### Fast capture (under 2 minutes)
1. Open a GitHub Issue using the **New Idea** template
2. Fill in: title, one sentence, layer, connection to existing ideas
3. Done — develop the full document later

### Full document
1. Choose the right layer folder
2. Copy the appropriate template from `_templates/`
3. Fill in front matter (id, title, layer, status, tags — minimum required)
4. Write at minimum: sections 1–4 (Problem, Vision, Philosophy, Mechanism)
5. The website reads from `data/ideas.json` — add via the site UI or sync manually

### The Solidarity Check
Every idea touching economics must answer:
> **"Where does solidarity happen structurally?"**
If redistribution requires voluntary action — redesign until it doesn't.

---

## Document Rules

- Front matter YAML block is required at the top of every document
- Every white paper must open with a `>` blockquote — one precise sentence
- Status lifecycle: `concept → research → active → (paused | archived)`

---

## File Structure

```
layer_folder/
└── project_name/
    ├── WHITE_PAPER.md     — philosophy, vision, mechanism
    ├── TECH_SPEC.md       — architecture, stack, code
    └── assets/            — diagrams, images (optional)
```
